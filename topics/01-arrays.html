Topic 01

Arrays & Two Pointers

The array is the foundation of every data structure. Understanding how memory layout, index access, and traversal patterns work unlocks every algorithm that follows.

GoogleMetaAmazonAppleMicrosoft

At a Glance

  • O(1) access by index — arrays store elements in contiguous memory. Index i lives at base_address + i * element_size. No traversal needed.
  • O(n) insert/delete at middle — inserting at index k shifts every element right of k by one position. Appending to the end is O(1) amortized.
  • Two-pointer technique — place L at index 0 and R at index n-1. Move them inward based on a condition. Turns O(n^2) brute force into O(n).
  • Sliding window — maintain a window of fixed or variable size. Slide right, adding one element and removing one. Finds subarray max/min/sum in O(n).
Core Concepts

Contiguous Memory

An array allocates a single block of memory. Element i is at address base + i * sizeof(T). This is why random access is O(1) — no pointer chasing, no cache misses from fragmentation.

Dynamic Arrays

When capacity is exhausted, the array doubles in size (typically 2x). The cost of doubling is O(n) but happens so rarely that amortized cost per append is O(1). Python lists and Java ArrayLists use this trick.

Two Pointers

Classic problems: two-sum on sorted array, container with most water, removing duplicates. Both pointers scan the array once total — O(n) instead of O(n^2) nested loops.

Sliding Window

Finding the maximum sum subarray of length k: slide a window right. Add the incoming element, subtract the outgoing. Single pass. Used in rate limiting, moving averages, and substring problems.

How It Works

Insert, Delete, Search

1

Insert at Index k

Shift all elements from index k to end one position right. Write the new value at k. Cost: O(n-k) shifts. Insert at end: O(1) amortized. Insert at front: O(n) always.

2

Delete at Index k

Shift all elements from k+1 to end one position left. Shrink the logical size by 1. Physical memory unchanged. Cost: O(n-k). Delete last element: O(1).

3

Linear Search

Scan from index 0 to n-1. Compare each element with the target. Best case O(1) (found at index 0). Worst case O(n). No assumption about order needed.

4

Two-Pointer Inward Scan

On a sorted array: place L=0, R=n-1. If arr[L]+arr[R] < target, move L right (need a larger value). If too large, move R left. Stop when L meets R. O(n) total moves.

5

Sliding Window

Fix window size k. Compute sum of first k elements. For each subsequent position: add arr[right], subtract arr[right-k]. One pass to find max-sum window.

Python
# Two-pointer: Two Sum on sorted array — O(n)
def two_sum(nums, target):
    L, R = 0, len(nums) - 1
    while L < R:
        s = nums[L] + nums[R]
        if s == target: return [L, R]
        elif s < target: L += 1
        else: R -= 1
    return []

# Sliding window: max sum of k consecutive — O(n)
def max_window_sum(nums, k):
    window = sum(nums[:k])
    best = window
    for i in range(k, len(nums)):
        window += nums[i] - nums[i - k]
        best = max(best, window)
    return best

Anti-patterns

Nested loops when two pointers work

Finding all pairs with a given sum in a sorted array in O(n^2) is common. If the array is sorted, two pointers solve it in O(n). Always ask: is the data ordered? Can I avoid the inner loop?

Recomputing window sums from scratch

Recalculating the sum of every window of size k takes O(nk). Sliding window reuses the previous sum — O(n). Recognize the overlap.

Forgetting index bounds after shift

After deleting element k and shifting left, the logical length is n-1. Accessing index n-1 reads garbage. Always track logical vs. physical size.

Interview Q&A

Common Interview Questions

A static array has a fixed size declared at creation time — you cannot grow it. A dynamic array (Python list, Java ArrayList, C++ vector) starts with some capacity and doubles when full. The doubling costs O(n) but happens so rarely that the amortized cost per append is O(1). Trade-off: dynamic arrays waste memory (up to 2x the elements) but provide O(1) amortized push.
Two pointers require the array to be sorted — then O(n) time, O(1) space. Hashmap works on unsorted arrays — O(n) time, O(n) space. Choose two pointers when space is constrained and sorting is cheap or already done. Choose hashmap when you cannot sort (order matters) or need to find indices.
Find the contiguous subarray with the largest sum. Kadane's: maintain cur = max(arr[i], cur + arr[i]) and best = max(best, cur). At each index, decide: extend the previous subarray or start fresh. O(n) time, O(1) space. Key insight: a subarray is worth extending only if the running sum is positive.
Start with capacity 1. Each time the array fills, copy all n elements to a new array of size 2n. Cost of copies across all inserts: 1+2+4+...+n = 2n. Spread over n inserts, that is 2 per insert — O(1) amortized. The doubling strategy ensures the total copy work is bounded by twice the total elements inserted.
