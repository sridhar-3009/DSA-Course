Topic 06

Hash Tables

O(1) average insert, lookup, and delete. The hash table is the single most useful data structure in interview problems — and understanding what makes it fast (and when it fails) separates good answers from great ones.

GoogleMetaAmazonStripeAirbnb

At a Glance

  • Hash function — maps a key to a bucket index. Good functions distribute keys uniformly, are fast to compute, and produce the same output for equal keys.
  • Collision handling — two keys mapping to the same bucket. Separate chaining (linked list per bucket) or open addressing (probe for next empty slot).
  • Load factor — n_keys / n_buckets. At ~0.75, resize by doubling buckets and rehashing all keys. Keeps chains short and probing fast.
  • Worst case O(n) — all keys hash to the same bucket. Avoided by good hash functions, random seeds, and resizing.
Core Concepts

Hash Function

Must be deterministic (same key = same hash), fast (O(key length)), and uniform (spread keys across buckets). Java uses object.hashCode(). Python uses hash(). Cryptographic hashes are too slow for hash tables.

Separate Chaining

Each bucket is a linked list. Collision: append to the list. Lookup: hash to bucket, scan list for key. Average O(1) with low load factor. Degrades to O(n) if all keys collide.

Open Addressing

Linear probing: on collision, check next slot. Quadratic probing: check i^2 steps ahead. Double hashing: second hash function determines step size. All keep data in the array — better cache behavior than chaining.

Resize & Rehash

When load factor exceeds threshold (0.75 typical), create a new array (2x size) and rehash every key into the new array. O(n) work, but amortized O(1) per operation.

Python
# HashMap patterns — most interview problems use these

# Frequency count
freq = {}
for c in "hello world":
    freq[c] = freq.get(c, 0) + 1

# Two-sum using hashmap — O(n) time, O(n) space
def two_sum(nums, target):
    seen = {}
    for i, n in enumerate(nums):
        complement = target - n
        if complement in seen:
            return [seen[complement], i]
        seen[n] = i

# Group anagrams — sort key trick
from collections import defaultdict
def group_anagrams(words):
    groups = defaultdict(list)
    for w in words:
        groups[tuple(sorted(w))].append(w)
    return list(groups.values())

Anti-patterns

Using mutable objects as keys

Hash tables require keys to be immutable (hashable). In Python, lists are not hashable — use tuples. In Java, if you override equals(), you must override hashCode() with the same fields, or the object will be "lost" in the map after mutation.

Assuming O(1) worst case

Hash collision attacks can deliberately degrade a hashmap to O(n). Python 3.3+ adds hash randomization by default. Java HashMap uses balanced trees in buckets when chain length exceeds 8.

Interview Q&A
Separate chaining: handles high load factors gracefully, simpler deletion, no clustering. Downside: extra memory for pointers, poor cache locality. Open addressing: better cache performance (data stays in the array), lower memory overhead. Downside: performance degrades sharply at high load factors, deletion requires tombstones (or backshifting). Use chaining when load factor may be high or deletions are frequent. Use open addressing when cache performance is critical and load factor is controlled.
Key decisions: (1) Hash function — key.hashCode() modulo array size. (2) Collision strategy — separate chaining (simpler) or linear probing (faster). (3) Load factor threshold — 0.75 is standard. (4) Resize factor — double on resize. (5) Handling deletions — tombstones for open addressing. Core operations: put(k,v): hash k, find bucket, update or append. get(k): hash k, find bucket, scan for k. remove(k): find and remove, mark tombstone if open addressing.
Use a hashmap: key = word, value = count. One pass through the text: freq[word] = freq.get(word, 0) + 1. O(n) time and space. To find the top k most frequent: use a min-heap of size k — for each (word, count) pair, if count > heap.min, replace. O(n log k). Alternatively, sort by frequency in O(n log n). The hashmap is the bottleneck on memory — counts for a large vocabulary fit in memory, but for a stream, use Count-Min Sketch.
