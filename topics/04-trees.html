Topic 04

Trees & BST

Trees are the most important non-linear structure in computer science. The binary search tree adds an ordering invariant that makes search, insert, and delete O(log n) on balanced trees.

GoogleMetaAmazonMicrosoft

At a Glance

  • BST invariant — every node: all values in left subtree < node.val, all values in right subtree > node.val. Enforced on insert, maintained through rotations on delete.
  • InOrder traversal = sorted order — traversing a BST inorder gives elements in ascending order. Used to validate BSTs and extract sorted sequences.
  • Height determines complexity — a balanced BST has height O(log n). A degenerate (sorted-insert) BST degrades to O(n) — effectively a linked list.
  • AVL / Red-Black trees — self-balancing BSTs. Rotations restore balance after insert/delete. Guarantee O(log n) worst case.
Core Concepts

Traversals

InOrder (L-Root-R): sorted output. PreOrder (Root-L-R): serialize tree, copy tree. PostOrder (L-R-Root): delete tree, evaluate expression tree. Level-order (BFS): shortest path, level-by-level processing.

BST Search

At each node: target == val: found. target < val: go left. target > val: go right. Pruning half the tree per step gives O(log n) on balanced trees.

BST Delete

Three cases: (1) leaf — just remove. (2) one child — replace with child. (3) two children — replace with inorder successor (smallest in right subtree), then delete successor from right subtree.

AVL Rotation

After insert, recompute heights bottom-up. If balance factor (left_height - right_height) exceeds 1 in magnitude, rotate. Left/right/left-right/right-left rotations restore balance in O(1).

Python
class TreeNode:
    def __init__(self, val):
        self.val=val; self.left=self.right=None

def insert(root, val):
    if not root: return TreeNode(val)
    if val < root.val: root.left = insert(root.left, val)
    else: root.right = insert(root.right, val)
    return root

def inorder(root, out=[]):
    if root:
        inorder(root.left, out)
        out.append(root.val)
        inorder(root.right, out)
    return out

# Validate BST — O(n)
def is_valid_bst(root, lo=float('-inf'), hi=float('inf')):
    if not root: return True
    if not (lo < root.val < hi): return False
    return (is_valid_bst(root.left, lo, root.val) and
            is_valid_bst(root.right, root.val, hi))

Anti-patterns

Inserting sorted data into a BST

Inserting [1,2,3,4,5] into a BST creates a right-skewed chain of height n. Every operation becomes O(n). Use a self-balancing BST (AVL, Red-Black) or shuffle input before inserting.

Forgetting the inorder successor has at most one child

The inorder successor of a node with two children (leftmost node in the right subtree) can only have a right child (never a left child — it is the minimum). Deleting it is always the simpler case 1 or 2.

Interview Q&A
A binary tree is any tree where each node has at most two children — no ordering constraint. A BST adds the invariant: left subtree values < node.val < right subtree values. This ordering enables O(log n) search. Binary heaps are also binary trees but use the heap property (parent <= children for min-heap), not BST ordering.
Pass valid range [lo, hi] through the recursion. Root has range (-inf, +inf). Left child of a node with value v gets range (lo, v). Right child gets (v, hi). If any node's value is outside its range, return false. This correctly handles cases where a "local" comparison passes but the global BST property is violated — a common mistake is checking only left.val < root.val < right.val without checking subtree ranges.
AVL trees are more strictly balanced (balance factor limited to 1) — faster lookups but more rotations on insert/delete. Red-Black trees allow slightly more imbalance — slightly slower lookups but fewer rotations. In practice: use Red-Black for write-heavy workloads (Linux kernel, Java TreeMap, C++ std::map use Red-Black). Use AVL for read-heavy workloads where lookup speed matters more than write speed.
