Topic 02

Linked Lists

A linked list trades O(1) random access for O(1) insert/delete at any known position. Every node points to the next — no contiguous memory required.

GoogleMetaAmazonMicrosoft

At a Glance

  • O(1) insert/delete at head — just update the head pointer. No shifting of other elements.
  • O(n) search — must traverse from head to find a node. No index arithmetic.
  • Doubly linked list — each node has prev and next pointers. Enables O(1) delete when you have a pointer to the node.
  • Floyd's cycle detection — slow pointer moves 1 step, fast moves 2. If there is a cycle, they will meet. O(n) time, O(1) space.
Core Concepts

Node Structure

Each node holds a value and a pointer (next). In a doubly linked list, also a prev pointer. Nodes live anywhere in memory — not contiguous. The head pointer is your entry point.

Insert at Head

Create a new node. Set new.next = head. Set head = new. Two pointer assignments — O(1) regardless of list length. No shifting.

Reverse a List

Keep three pointers: prev = null, cur = head, next = cur.next. Flip cur.next = prev, then advance all three. One pass — O(n).

Slow/Fast Pointer

Slow moves +1 each step, fast moves +2. If a cycle exists, fast laps slow — they meet inside the cycle. Also finds the middle of a list (slow stops at middle when fast reaches end).

How It Works

Operations Step by Step

1

Insert at Head — O(1)

new.next = head; head = new. That is it. The rest of the list is untouched. Cost is the same whether the list has 5 nodes or 5 million.

2

Insert at Tail — O(n)

Without a tail pointer: traverse to the last node, then new.last.next = new. With a tail pointer: O(1). Always maintain a tail pointer for queues and append-heavy workloads.

3

Delete by Value — O(n)

Traverse to find the node whose next.val equals target. Set cur.next = cur.next.next. The garbage collector (or free()) handles the unlinked node. Edge case: deleting the head.

4

Reverse — O(n)

Three pointers traverse once. At each step: save next = cur.next, flip cur.next = prev, advance prev = cur, advance cur = next. After the loop, set head = prev.

5

Cycle Detection (Floyd's)

Initialize slow = fast = head. While fast != null and fast.next != null: slow = slow.next; fast = fast.next.next. If slow == fast: cycle detected. Why it works: fast gains 1 step per iteration; in a cycle of length c, it catches up in at most c iterations.

Python
class Node:
    def __init__(self, val):
        self.val = val
        self.next = None

# Reverse a singly linked list — O(n)
def reverse(head):
    prev = None
    cur = head
    while cur:
        nxt = cur.next
        cur.next = prev
        prev = cur
        cur = nxt
    return prev

# Floyd's cycle detection — O(n) time O(1) space
def has_cycle(head):
    slow = fast = head
    while fast and fast.next:
        slow = slow.next
        fast = fast.next.next
        if slow is fast: return True
    return False

*Singly LL delete a known node

If you only have a pointer to the node (not its predecessor), you cannot delete it in O(1) with a singly linked list. Trick: copy the next node's value into the current node, then delete the next node. Works unless the node is the tail.

Anti-patterns

No tail pointer on append-heavy lists

Appending to a singly linked list without a tail pointer requires traversing to the last node every time — O(n) per append. Maintain a tail pointer; update it on every insert-at-tail and delete-tail.

Forgetting edge cases in reversal

Reversing must handle: empty list (return null), single node (return it), and the tail becoming the new head. Failing to set old_head.next = null creates a cycle.

Interview Q&A
Choose a linked list when: (1) you frequently insert/delete at the front or middle and have a pointer to the position; (2) you do not need random access by index; (3) the size is unknown and you want to avoid reallocation. Choose an array when you need O(1) index access, better cache performance (contiguous memory), or binary search.
Use slow and fast pointers both starting at head. Move slow one step, fast two steps per iteration. When fast reaches the end (fast == null or fast.next == null), slow is at the middle. For even-length lists, slow lands on the second-middle node.
After Floyd's algorithm detects a meeting point inside the cycle, reset one pointer to head. Move both pointers one step at a time. They meet at the cycle's entry node. Mathematical proof: if fast traveled 2d and slow traveled d steps to meet, and the cycle starts k steps from head, then the meeting point is k steps from the cycle start — same as the head-to-cycle-start distance.
