Topic 08

Sorting Algorithms

Six sorting algorithms. Each makes different trade-offs between comparisons, swaps, memory, and stability. Knowing which to use — and why — is a fundamental engineering skill.

GoogleMetaAmazonMicrosoftApple

At a Glance

  • Comparison sort lower bound — any algorithm that sorts by comparing elements needs at least O(n log n) comparisons in the worst case. This is provably optimal.
  • Quicksort in practice — O(n log n) average, O(n^2) worst case (sorted input + bad pivot). Random pivot or median-of-three avoids worst case. Best cache performance in practice.
  • Merge sort — O(n log n) guaranteed, stable, but requires O(n) extra space. Standard for linked lists and external sorting.
  • Stability — a stable sort preserves the relative order of equal elements. Merge sort is stable; quicksort (standard) is not.
Core Concepts

Bubble Sort

Compare adjacent pairs, swap if out of order. Repeat n times. O(n^2) comparisons and swaps. Only useful for nearly sorted data (O(n) with early termination). Stable.

Merge Sort

Divide array in half recursively until size 1. Merge two sorted halves — O(n) per merge. T(n) = 2T(n/2) + O(n) = O(n log n). Stable. O(n) space. Best for linked lists.

Quick Sort

Choose pivot. Partition: elements < pivot left, > pivot right. Recurse. Average O(n log n), worst O(n^2). Not stable. In-place. Best cache performance — widely used in practice.

Counting / Radix Sort

Non-comparison sorts. Counting: O(n+k) where k = range of values. Radix: O(d(n+k)) where d = digits. Break the O(n log n) barrier when element range is limited.

Python
# Merge sort — O(n log n), stable, O(n) space
def merge_sort(arr):
    if len(arr) <= 1: return arr
    mid = len(arr) // 2
    L = merge_sort(arr[:mid])
    R = merge_sort(arr[mid:])
    result, i, j = [], 0, 0
    while i < len(L) and j < len(R):
        if L[i] <= R[j]: result.append(L[i]); i += 1
        else: result.append(R[j]); j += 1
    return result + L[i:] + R[j:]

# Quick sort — O(n log n) avg, in-place
def quick_sort(arr, lo, hi):
    if lo >= hi: return
    pivot = arr[hi]
    i = lo - 1
    for j in range(lo, hi):
        if arr[j] <= pivot:
            i += 1
            arr[i], arr[j] = arr[j], arr[i]
    arr[i+1], arr[hi] = arr[hi], arr[i+1]
    quick_sort(arr, lo, i)
    quick_sort(arr, i+2, hi)

Interview Q&A
Insertion sort excels on small arrays (n < 20) and nearly-sorted arrays. O(n) on sorted input (best case). Low overhead — no recursion, no extra space. Java's Arrays.sort and Python's Timsort use insertion sort for small subarrays within their merge sort implementations. Also useful in online settings where elements arrive one at a time.
Timsort (Tim Peters, 2002) is a hybrid of merge sort and insertion sort. It finds already-sorted "runs" in the input (exploiting natural ordering), sorts small runs with insertion sort, then merges runs. O(n) on nearly sorted data, O(n log n) worst case. Stable. This pattern is extremely common in real-world data — database results, log files, user input. Python's list.sort() and Java's Arrays.sort() for objects use Timsort.
