Topic 07

Graphs

Graphs are the most general structure — trees are a special case. They model networks, dependencies, maps, and social connections. BFS, DFS, and Dijkstra are the three algorithms you must know cold.

GoogleMetaUberLinkedInAmazon

At a Glance

  • BFS — explores level by level using a queue. Finds shortest path (fewest edges) in unweighted graphs. O(V+E).
  • DFS — explores as deep as possible using a stack (or recursion). Detects cycles, topological sort, connected components. O(V+E).
  • Dijkstra — shortest path in weighted graphs with non-negative edges. Uses a min-heap. O((V+E) log V).
  • Topological sort — linear ordering of vertices such that every directed edge u-v has u before v. Only valid for DAGs. Used in build systems, course scheduling.
Core Concepts

Adjacency List vs Matrix

Adjacency list: O(V+E) space, O(degree) to find neighbors — sparse graphs. Adjacency matrix: O(V^2) space, O(1) edge lookup — dense graphs. Most real graphs are sparse — use adjacency list.

BFS Shortest Path

BFS explores all nodes at distance d before distance d+1. When the target is first visited, the path found is guaranteed to be the shortest (fewest edges). Does not work for weighted edges.

Dijkstra

Greedy: always relax the unvisited node with the smallest known distance. Min-heap tracks the frontier. Cannot handle negative edge weights (use Bellman-Ford instead).

Union-Find

Tracks connected components. Union: merge two components. Find: determine component of a node. With path compression and union by rank: nearly O(1) amortized per operation.

Python
from collections import deque
import heapq

# BFS shortest path — O(V+E)
def bfs(graph, start, end):
    queue = deque([[start]])
    visited = {start}
    while queue:
        path = queue.popleft()
        node = path[-1]
        if node == end: return path
        for nb in graph[node]:
            if nb not in visited:
                visited.add(nb)
                queue.append(path + [nb])
    return []

# Dijkstra — O((V+E) log V)
def dijkstra(graph, start):
    dist = {node: float('inf') for node in graph}
    dist[start] = 0
    heap = [(0, start)]
    while heap:
        d, u = heapq.heappop(heap)
        if d > dist[u]: continue
        for v, w in graph[u]:
            if dist[u] + w < dist[v]:
                dist[v] = dist[u] + w
                heapq.heappush(heap, (dist[v], v))
    return dist

Anti-patterns

Not marking nodes visited in BFS/DFS

Without a visited set, BFS/DFS will loop forever on a graph with cycles. Always mark a node visited when it is first enqueued (BFS) or when first entered (DFS) — not after processing.

Using Dijkstra with negative edges

Dijkstra assumes that once a node is extracted from the heap, its shortest distance is finalized. A negative edge can create a shorter path to an already-visited node. Use Bellman-Ford or SPFA for negative edges.

Interview Q&A
DFS with three states per node: unvisited (white), in-current-path (gray), fully-done (black). If you reach a gray node during DFS, you found a back edge — a cycle exists. Alternatively, topological sort (Kahn's algorithm): if not all nodes are in the topological order, there is a cycle.
BFS: unit-weight edges, queue, O(V+E). Dijkstra: non-negative weights, min-heap, O((V+E) log V). Bellman-Ford: any weights including negative, relaxes all edges V-1 times, O(VE). Also detects negative cycles. Dijkstra is faster but restricted to non-negative weights. Bellman-Ford is slower but handles negative edges. BFS is fastest but only for unweighted graphs.
Union-Find (Disjoint Set Union) tracks connected components. Find(x): returns the root of x's component. Union(x,y): merges x and y's components. With path compression (flatten tree on find) and union by rank (attach smaller tree under larger): near-O(1) amortized per operation (inverse Ackermann). Use cases: Kruskal's MST, detecting cycles in undirected graphs, number of connected components, network connectivity queries.
