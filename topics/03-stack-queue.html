Topic 03

Stacks & Queues

Two restricted data structures that turn up everywhere — from browser history to BFS traversal. Their constraints (LIFO vs FIFO) are features, not limitations.

GoogleMetaAmazonMicrosoftApple

At a Glance

  • Stack (LIFO) — last in, first out. Push to top, pop from top. Used for DFS, undo, call stack, expression evaluation.
  • Queue (FIFO) — first in, first out. Enqueue at rear, dequeue from front. Used for BFS, task scheduling, message queues.
  • Monotonic stack — maintain elements in increasing/decreasing order. Solves "next greater element" problems in O(n).
  • Deque — double-ended queue. O(1) push/pop at both ends. Implements sliding window maximum in O(n).
Core Concepts

Stack

Push: add to top. Pop: remove from top. Peek: read top without removing. All O(1). Implemented with an array or linked list. The call stack in your CPU is a hardware stack.

Queue

Enqueue: add to rear. Dequeue: remove from front. Both O(1) with a doubly linked list or circular array. Circular array avoids the O(n) shift after each dequeue.

Monotonic Stack

Keep popping while the top violates a monotonic property. Example: "next greater element" — for each element, pop everything smaller and they all found their answer. O(n) total.

Balanced Brackets

Push opening brackets. On closing bracket: if top matches, pop; else invalid. If stack is empty at end: valid. Classic interview problem demonstrating stack's LIFO property.

How It Works

Stack and Queue Operations

1

Stack Push — O(1)

Increment top pointer. Write value at arr[top]. If using a linked list: create node, set node.next = head, set head = node. Both are constant time.

2

Stack Pop — O(1)

Read arr[top]. Decrement top pointer. The memory is not erased — just logically excluded. Stack underflow (popping empty stack) must be guarded against explicitly.

3

Queue Enqueue — O(1)

Add to rear. With a circular array: rear = (rear + 1) % capacity. With a linked list: update tail.next and tail pointer. Never O(n).

4

Queue Dequeue — O(1)

Read from front. Advance front = (front + 1) % capacity. Circular array reuses slots. Without circular indexing, naive arrays dequeue in O(n) due to shifting.

5

Balanced Brackets Algorithm

For each character: if opening bracket (([{), push it. If closing bracket: check if stack top is the matching opener. If yes, pop; if no or stack empty, return false. After all characters, return stack.isEmpty().

Python
# Balanced brackets — O(n) time, O(n) space
def is_valid(s):
    stack = []
    pairs = {')': '(', ']': '[', '}': '{'}
    for c in s:
        if c in '([{':
            stack.append(c)
        elif not stack or stack[-1] != pairs[c]:
            return False
        else:
            stack.pop()
    return not stack

# Monotonic stack: next greater element — O(n)
def next_greater(nums):
    result = [-1] * len(nums)
    stack = []
    for i, v in enumerate(nums):
        while stack and nums[stack[-1]] < v:
            result[stack.pop()] = v
        stack.append(i)
    return result

Anti-patterns

Using a list as a queue (O(n) dequeue)

In Python, list.pop(0) is O(n) — all elements shift left. Use collections.deque which is O(1) at both ends. In interviews, always clarify whether the queue is array-backed (naive) or linked/circular before discussing complexity.

Forgetting to handle empty stack on pop

Popping an empty stack is undefined behavior in C++ and throws in Python/Java. Every production pop should check if not stack or catch the exception. In bracket validation, an empty stack on a closing bracket means invalid — don't pop blindly.

Interview Q&A
Maintain two stacks: inbox and outbox. Enqueue: push to inbox. Dequeue: if outbox is empty, pop everything from inbox onto outbox (reversing the order), then pop from outbox. Amortized O(1) per operation — each element is moved at most once from inbox to outbox.
A monotonic stack maintains elements in strictly increasing or decreasing order. When a new element violates the order, pop elements until the order is restored. Each element is pushed and popped at most once — O(n) total. Use cases: next greater element, largest rectangle in histogram, trapping rainwater, daily temperatures.
Maintain a second "min stack" in parallel. On push(v): also push min(v, min_stack.top()) onto the min stack. On pop: pop from both stacks simultaneously. getMin() peeks the min stack — O(1). Space: O(n). Alternative: store (value, current_min) as pairs in a single stack.
