Topic 05

Heaps & Priority Queues

A heap gives you O(log n) insert and O(log n) extract-min (or max) — with O(1) peek at the minimum. It is the most efficient data structure for repeatedly pulling the next priority item.

GoogleAmazonMicrosoftMeta

At a Glance

  • Complete binary tree stored as array — parent of index i is at (i-1)/2. Children at 2i+1 and 2i+2. No pointers needed.
  • Heap property — min-heap: parent <= both children. Max-heap: parent >= both children. Only between parent and child — siblings have no required order.
  • Insert: bubble up — add to end, swap with parent while heap property is violated. At most O(log n) swaps.
  • Extract-min: bubble down — replace root with last element, remove last, swap root down with smaller child. O(log n) swaps.
Core Concepts

Array Representation

For index i: parent = floor((i-1)/2), left child = 2i+1, right child = 2i+2. A complete binary tree stored in level-order in an array — cache-friendly with no pointer overhead.

Heapify — O(n)

Building a heap from an unordered array in O(n) (not O(n log n)). Start from the last non-leaf (n/2 - 1) and bubble down each node. Lower levels do less work — the math yields a linear sum.

K Largest Elements

Maintain a min-heap of size k. For each element: if larger than heap min, pop the min and push the new element. Final heap contains k largest. O(n log k) — better than O(n log n) sort when k << n.

Heap Sort

Build a max-heap in O(n). Extract max n times, each O(log n). Total: O(n log n). In-place, not stable. Practically slower than quicksort due to cache unfriendliness (jumps around the array).

Python
import heapq

# Min-heap (Python's default)
h = []
heapq.heappush(h, 5)
heapq.heappush(h, 1)
heapq.heappush(h, 3)
print(heapq.heappop(h))  # 1 — always the minimum

# K largest elements — O(n log k)
def k_largest(nums, k):
    heap = nums[:k]
    heapq.heapify(heap)              # O(k)
    for n in nums[k:]:
        if n > heap[0]:
            heapq.heapreplace(heap, n)   # O(log k)
    return heap

# Merge k sorted lists — O(n log k)
def merge_k_sorted(lists):
    h = [(lst[0], i, 0) for i, lst in enumerate(lists) if lst]
    heapq.heapify(h)
    result = []
    while h:
        val, i, j = heapq.heappop(h)
        result.append(val)
        if j + 1 < len(lists[i]):
            heapq.heappush(h, (lists[i][j+1], i, j+1))
    return result

Interview Q&A
A heap has the heap property (parent <= children for min-heap) but siblings have no required order. A BST has the BST property (left < node < right) which gives total ordering along any path. Heap: O(1) min, O(log n) insert/extract. BST: O(log n) search/insert/delete, but supports in-order traversal and range queries. Use heap for priority queues; use BST for sorted data with arbitrary search.
Maintain two heaps: a max-heap for the lower half of numbers, and a min-heap for the upper half. Keep them balanced (sizes differ by at most 1). For each new number: if <= max-heap top, push to max-heap; otherwise push to min-heap. Rebalance if sizes differ by more than 1. Median is the top of the larger heap, or the average of both tops if equal size. O(log n) insert, O(1) median.
