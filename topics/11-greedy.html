Topic 11

Greedy Algorithms

Greedy algorithms make the locally optimal choice at each step, hoping to reach the globally optimal solution. When they work, they are fast and elegant. When they do not, they give wrong answers — knowing the difference is the skill.

GoogleAmazonMetaUber

At a Glance

  • Greedy choice property — a globally optimal solution can be reached by making locally optimal choices. Must be proven (not assumed) for each problem.
  • Optimal substructure — the optimal solution to the problem contains optimal solutions to subproblems. Shared with DP — but greedy requires the greedy choice to be safe without exploring alternatives.
  • When greedy fails — coin change with non-standard denominations (e.g., [10,7,1], target=14: greedy gives 10+1+1+1=4 coins; DP gives 7+7=2 coins). Always verify greedy correctness.
  • Classic greedy problems — activity selection, fractional knapsack, Huffman coding, Dijkstra, Prim's MST, Kruskal's MST, interval scheduling.
Core Concepts

Activity Selection

Select maximum non-overlapping intervals. Greedy: always pick the interval with the earliest finish time. Provably optimal — finishing earlier leaves more room for future intervals. O(n log n) to sort, O(n) to select.

Fractional Knapsack

Items can be split. Greedy: sort by value/weight ratio (descending). Take as much of the highest-ratio item as possible. O(n log n). Unlike 0/1 knapsack, greedy works here because fractions allow exact capacity fill.

Huffman Coding

Build optimal prefix-free code. Greedy: always merge the two lowest-frequency nodes into a new internal node. Repeat until one tree. Guarantees minimum total encoded length.

Interval Scheduling

Variant: minimum number of platforms/rooms needed. Sort start and end times separately. Two-pointer scan — when a train arrives (start[i]), increment platforms; when one departs (end[j]), decrement. Peak is the answer.

Python
# Activity selection — O(n log n)
def activity_selection(activities):
    # activities: list of (start, end)
    sorted_acts = sorted(activities, key=lambda x: x[1])
    selected = [sorted_acts[0]]
    for act in sorted_acts[1:]:
        if act[0] >= selected[-1][1]:
            selected.append(act)
    return selected

# Jump game — can reach end? — O(n)
def can_jump(nums):
    reach = 0
    for i, v in enumerate(nums):
        if i > reach: return False
        reach = max(reach, i + v)
    return True

# Minimum platforms needed — O(n log n)
def min_platforms(arrivals, departures):
    arrivals.sort(); departures.sort()
    platforms = result = i = j = 0
    while i < len(arrivals):
        if arrivals[i] < departures[j]:
            platforms += 1; i += 1
        else:
            platforms -= 1; j += 1
        result = max(result, platforms)
    return result

Anti-patterns

Assuming greedy always works for optimization

Greedy works for activity selection and fractional knapsack but not for 0/1 knapsack, coin change with arbitrary denominations, or shortest paths with negative weights. Always ask: does the greedy choice provably not close off better future choices?

Forgetting to sort

Most greedy algorithms require pre-sorting by a key (end time, ratio, weight). A greedy algorithm that does not sort is either wrong or not truly greedy — it is just a heuristic.

Interview Q&A
Two main proof techniques: (1) Exchange argument — take any optimal solution. Show you can swap its first choice with the greedy choice without making things worse. Repeat until the optimal solution matches the greedy solution. (2) Greedy stays ahead — inductively prove that after each greedy step, the greedy solution is at least as good as any other solution at the same step. Without a proof, a greedy algorithm is just a heuristic.
Given an array where nums[i] = max jump from index i, can you reach the last index? Greedy: track the furthest index reachable. For each index i: if i > reach, you cannot get here — return false. Otherwise: reach = max(reach, i + nums[i]). Return true. Greedy is optimal because the maximum reachable index is a monotonic property — extending reach never hurts you. O(n) time, O(1) space.
