Topic 09

Binary Search

Binary search turns O(n) linear scan into O(log n) by exploiting sorted order. More importantly, the binary search pattern applies to any problem where the search space can be halved each iteration.

GoogleMetaAmazonMicrosoft

At a Glance

  • Precondition: sorted array — binary search requires a monotonic search space. Without sorted order, you cannot safely eliminate half the elements.
  • O(log n) comparisons — each step halves the search space. Searching 1 billion elements takes at most 30 comparisons.
  • Off-by-one errors — the most common bug. Know two templates: [lo, hi] closed interval (L<=R) vs [lo, hi) half-open (L<R). Pick one and use it consistently.
  • Binary search on answer — "find minimum x such that f(x) is true" — apply binary search on the value range, not the array index. Powerful generalization.
Core Concepts

Standard Binary Search

L=0, R=n-1. While L<=R: mid=(L+R)/2. If arr[mid]==target: found. If arr[mid]<target: L=mid+1. Else: R=mid-1. Terminates in O(log n). Use (L+R)//2, not (L+R)/2 in languages without integer division.

Lower Bound

Find leftmost index where arr[i] >= target. When arr[mid] >= target: R=mid-1 (do not stop, search left). When arr[mid] < target: L=mid+1. Return L. Used for insertion position.

Binary Search on Answer

Is it possible to ship all packages in D days? Binary search D between [max_weight, total_weight]. Check feasibility in O(n). Total: O(n log n). Pattern: minimize/maximize a value subject to a constraint.

Rotated Sorted Array

Array [4,5,6,7,0,1,2] — sorted, then rotated. Binary search still works: determine which half is sorted (compare with lo/hi), then check if target is in the sorted half. O(log n).

Python
# Standard binary search — O(log n)
def binary_search(nums, target):
    L, R = 0, len(nums) - 1
    while L <= R:
        mid = (L + R) // 2
        if nums[mid] == target: return mid
        elif nums[mid] < target: L = mid + 1
        else: R = mid - 1
    return -1

# Binary search on answer: capacity to ship in D days
def ship_in_days(weights, days):
    def feasible(cap):
        ships, cur = 1, 0
        for w in weights:
            if cur + w > cap: ships += 1; cur = 0
            cur += w
        return ships <= days
    L, R = max(weights), sum(weights)
    while L < R:
        mid = (L + R) // 2
        if feasible(mid): R = mid
        else: L = mid + 1
    return L

Interview Q&A
After rotation, one half of the array is always sorted. At each step: compare arr[mid] with arr[lo]. If arr[lo] <= arr[mid], the left half is sorted — check if target is in [arr[lo], arr[mid]]; if yes, search left; else search right. Otherwise, the right half is sorted — similar logic. O(log n). Key insight: always determine which half is sorted first.
Problem: find the minimum capacity C to ship all packages within D days. As C increases, it becomes easier to ship in D days — monotonic. Binary search C between [max_weight, sum_of_weights]. For each candidate C, simulate the shipping in O(n). Total: O(n log(sum)). Other examples: "minimum days to complete tasks", "koko eating bananas", "split array largest sum" — all monotonic feasibility problems.
