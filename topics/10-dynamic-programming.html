Topic 10

Dynamic Programming

Dynamic programming is not a data structure or an algorithm — it is a design technique. It turns exponential recursion into polynomial computation by storing the results of overlapping subproblems.

GoogleMetaAmazonMicrosoftNetflix

At a Glance

  • Overlapping subproblems — the same subproblem is computed multiple times. DP stores (memoizes) results to avoid redundant work.
  • Optimal substructure — the optimal solution to a problem contains optimal solutions to its subproblems. Required for DP to work.
  • Top-down vs bottom-up — top-down: recursion + memoization. Bottom-up: fill a table iteratively from base cases up. Same complexity, different trade-offs.
  • State definition is everything — the hardest part is defining what dp[i] (or dp[i][j]) represents. Get that right and the recurrence follows.
Core Concepts

Memoization (Top-Down)

Write the recursive solution. Add a cache (dict/array). Before computing, check cache. After computing, store result. Keeps the recursive structure — easier to think about. Risk: stack overflow on deep recursion.

Tabulation (Bottom-Up)

Fill a table from base cases upward. No recursion — no stack overflow. Usually slightly faster (no function call overhead). Requires determining the correct fill order.

1D DP Patterns

Fibonacci, climbing stairs, coin change, word break, longest increasing subsequence. State: dp[i] = answer for first i elements. Transition: dp[i] = f(dp[i-1], dp[i-2], ...).

2D DP Patterns

Edit distance, longest common subsequence, knapsack, matrix chain multiplication. State: dp[i][j] = answer considering i items with capacity j. Transition considers including or excluding each item.

Python
# Fibonacci — 3 approaches
def fib_naive(n):          # O(2^n)
    if n <= 1: return n
    return fib_naive(n-1) + fib_naive(n-2)

def fib_memo(n, memo={}):  # O(n)
    if n in memo: return memo[n]
    if n <= 1: return n
    memo[n] = fib_memo(n-1) + fib_memo(n-2)
    return memo[n]

# Coin change — O(amount * len(coins))
def coin_change(coins, amount):
    dp = [float('inf')] * (amount + 1)
    dp[0] = 0
    for a in range(1, amount + 1):
        for c in coins:
            if c <= a:
                dp[a] = min(dp[a], dp[a-c] + 1)
    return dp[amount] if dp[amount] != float('inf') else -1

Interview Q&A
(1) Identify the state: what information do you need to represent a subproblem? (2) Define the recurrence: how does the answer for state (i) depend on smaller states? (3) Identify base cases. (4) Determine fill order (tabulation) or confirm acyclic dependencies (memoization). (5) Optimize space if the recurrence only looks back a fixed number of states.
Coin change allows unlimited use of each coin — it is an unbounded knapsack. 0/1 knapsack allows each item at most once. In the recurrence, unbounded: dp[i] = min(dp[i], dp[i-coin]+1) where you can reuse the same coin. 0/1: iterate items in the outer loop, capacity in the inner loop, and update the table going forward (unbounded) or backward (0/1 — prevents reuse).
Use a 1D array dp[0..W]. For 0/1 knapsack: iterate capacity from W down to item_weight (right to left). This ensures each item is only used once — when you update dp[w], dp[w - item.w] has not been updated yet in this iteration. For unbounded knapsack: iterate left to right — dp[w - coin] is already updated, allowing reuse.
